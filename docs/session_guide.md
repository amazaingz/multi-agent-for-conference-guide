# re:Invent 议程规划指南

## Session 类型介绍

### 1. Keynote（主题演讲）
- **时长**: 2-3 小时
- **特点**: 重大产品发布、战略方向
- **建议**: 必须参加，无需注册
- **主要 Keynote**:
  - Monday Night Live (CEO Keynote)
  - Tuesday Morning Keynote (新产品发布)
  - Wednesday Morning Keynote (技术深度)

### 2. Breakout Session（分组会议）
- **时长**: 45-60 分钟
- **级别**: 
  - 100 level: 入门
  - 200 level: 中级
  - 300 level: 高级
  - 400 level: 专家
- **特点**: 深入技术主题
- **建议**: 根据兴趣和技术水平选择

### 3. Workshop（工作坊）
- **时长**: 2-4 小时
- **特点**: 动手实践，需要笔记本电脑
- **建议**: 需要提前注册，名额有限
- **准备**: 提前配置 AWS 账号

### 4. Chalk Talk（圆桌讨论）
- **时长**: 45-60 分钟
- **特点**: 小组讨论，互动性强
- **建议**: 适合深入交流特定话题

### 5. Builders' Session（构建者会议）
- **时长**: 60 分钟
- **特点**: 小班教学，动手实践
- **建议**: 需要提前注册，名额很少

## 热门主题分类

### AI/ML（人工智能/机器学习）
- **服务**: SageMaker, Bedrock, Q Developer
- **推荐 Session**:
  - Generative AI 应用开发
  - ML 模型训练与部署
  - AI 安全与治理

### Serverless（无服务器）
- **服务**: Lambda, API Gateway, Step Functions
- **推荐 Session**:
  - Serverless 架构最佳实践
  - Event-driven 应用
  - 成本优化

### Containers & Kubernetes
- **服务**: ECS, EKS, Fargate
- **推荐 Session**:
  - 容器安全
  - Kubernetes 最佳实践
  - 微服务架构

### Database（数据库）
- **服务**: RDS, DynamoDB, Aurora
- **推荐 Session**:
  - 数据库迁移
  - 性能优化
  - 多区域部署

### Security（安全）
- **服务**: IAM, GuardDuty, Security Hub
- **推荐 Session**:
  - 零信任架构
  - 合规性
  - 威胁检测

### Networking（网络）
- **服务**: VPC, CloudFront, Route 53
- **推荐 Session**:
  - 混合云网络
  - 全球加速
  - 网络安全

## 议程规划建议

### Day 1 (Monday)
- **上午**: 注册、领取徽章
- **下午**: 探索 Expo、参加 Breakout Session
- **晚上**: Monday Night Live Keynote

### Day 2 (Tuesday)
- **上午**: Tuesday Morning Keynote
- **下午**: Breakout Sessions / Workshop
- **晚上**: Partner Events / Networking

### Day 3 (Wednesday)
- **上午**: Wednesday Morning Keynote
- **下午**: Breakout Sessions / Chalk Talks
- **晚上**: re:Play Party

### Day 4 (Thursday)
- **全天**: Breakout Sessions / Workshops
- **下午**: Expo 最后机会
- **晚上**: 自由活动

### Day 5 (Friday)
- **上午**: 最后的 Sessions
- **中午**: 离场

## 选择 Session 的技巧

### 1. 提前规划
- 使用 re:Invent Mobile App
- 提前 2-3 周开始规划
- 标记感兴趣的 Session

### 2. 优先级排序
- **必须参加**: Keynote、关键技术 Session
- **希望参加**: 感兴趣的主题
- **备选**: 时间允许的情况下

### 3. 避免冲突
- 同一时间段不要安排太多
- 预留会场间移动时间（15-20 分钟）
- 考虑会场位置

### 4. 平衡安排
- 技术深度与广度
- 理论与实践
- 学习与社交

### 5. 灵活调整
- 有些 Session 可能满员
- 现场可能有更感兴趣的内容
- 注意身体状况，适当休息

## Session 注册技巧

### 提前注册
- **开放时间**: 通常在会议前 3-4 周
- **热门 Session**: 开放后立即注册
- **Workshop**: 名额最少，最难抢

### Waitlist（候补名单）
- 加入候补名单
- 会前可能有人取消
- 现场也可以尝试 Walk-in

### 现场注册
- 部分 Session 接受现场注册
- 提前 15-30 分钟到场
- 准备备选方案

## 会场分布

### 主要会场
1. **Venetian / Palazzo**: 最大会场，大部分 Session
2. **Caesars Forum**: 新会场，现代化设施
3. **Wynn / Encore**: 高端会场，部分 Session
4. **MGM Grand**: Expo 和部分 Session
5. **Mandalay Bay**: 部分 Session

### 会场间交通
- **步行**: 5-20 分钟（会场间）
- **Shuttle Bus**: 免费班车
- **Taxi/Uber**: 高峰期可能拥堵

### 时间规划
- 预留 15-20 分钟移动时间
- 考虑会场内部也很大
- 避免连续安排不同会场的 Session

- 适当休息

### 笔记整理

- 拍照保存 Slides
- 及时整理笔记

## 常见问题

### Q: Session 满员怎么办？
A: 加入 Waitlist，或现场尝试 Walk-in，或观看会后录像

### Q: 如何选择 Session 级别？
A: 根据自己的技术水平，200-300 level 适合大多数人

### Q: Workshop 需要什么准备？
A: 笔记本电脑、AWS 账号、相关技术基础

### Q: 可以中途离开 Session 吗？
A: 可以，但尽量不要打扰他人

### Q: Session 有中文翻译吗？
A: 部分 Session 有，但大多数是英文
